User Management API Documentation

Base URL

http://localhost:8080/api

Authentication Endpoints

  • POST /api/auth/register - Register new user
  • POST /api/auth/login - Login and get JWT token
  • POST /api/auth/logout - Logout and invalidate token Auth Required
  • POST /api/auth/refresh - Refresh JWT token Auth Required
  • POST /api/auth/verify-email - Verify email address
  • POST /api/auth/forgot-password - Request password reset
  • POST /api/auth/reset-password - Reset password with token

User Endpoints

  • GET /api/users/profile - Get current user profile Auth Required
  • PUT /api/users/profile - Update profile information Auth Required
  • POST /api/users/change-password - Change password Auth Required
  • DELETE /api/users/account - Delete own account Auth Required

Admin Endpoints

  • GET /api/admin/users - List all users with pagination Admin Required
  • GET /api/admin/users/{id} - Get user by ID Admin Required
  • PUT /api/admin/users/{id}/status - Update user status (active/inactive) Admin Required
  • PUT /api/admin/users/{id}/role - Update user role Admin Required
  • DELETE /api/admin/users/{id} - Delete user account Admin Required
  • GET /api/admin/stats - Get user statistics Admin Required

Support Ticket Endpoints

  • POST /api/support/tickets - Create new support ticket Auth Required
  • GET /api/support/tickets - Get current user's tickets Auth Required
  • GET /api/support/tickets/{id} - Get specific ticket details Auth Required
  • PUT /api/support/tickets/{id} - Update ticket (user can update description) Auth Required
  • PUT /api/support/tickets/{id}/status - Update ticket status Admin Required

Authentication

For protected endpoints, include the JWT token in the Authorization header. Tokens expire after 24 hours, use the refresh endpoint to get a new token.

Authorization: Bearer YOUR_JWT_TOKEN
